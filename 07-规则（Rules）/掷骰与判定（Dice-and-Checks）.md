# 掷骰与判定（Dice & Checks）

## 掷骰分工

- 玩家角色公开检定：默认由玩家掷骰并报告原始骰点；玩家要求时可以由 Agent 代掷。
- 玩家角色攻击和伤害：默认由玩家掷骰；玩家要求时可以由 Agent 一次性完成。
- 玩家角色死亡豁免：默认由玩家掷骰。
- 隐藏检定：由 Agent 掷骰。
- 敌方和 NPC 检定：由 Agent 掷骰。
- 随机遭遇、天气及秘密事件：由 Agent 掷骰。
- 是否展示原始骰、加值和总值：
  - 公开检定：展示完整公式；
  - 战斗检定：展示完整公式；
  - 隐藏检定：当场只说明可观察结果，具体公式记录在会话日志；
  - 纯叙事随机表：无需逐项展示，除非玩家询问。

公开检定格式：

`1d20（原始骰）+ 属性调整值 + 熟练加值 + 其他修正 = 总值，对比 DC`

示例：

`1d20（14）+3 力量 +2 熟练 = 19，对比 DC 15，成功。`

## 玩家自行掷骰

玩家可以使用：

- 实体骰子；
- 本地骰点程序；
- Agent 提供的骰点工具；
- 其他能够产生随机结果的方式。

玩家报告骰点后，Agent 默认接受该结果，不重复代掷。

如果玩家只报告原始 d20 点数，Agent 负责根据角色文件计算加值。

## 何时要求检定

只有同时满足以下条件时才要求掷骰：

1. 行动结果存在真实不确定性；
2. 成功与失败会产生不同后果；
3. 失败具有实际意义；
4. 当前行动没有被规则明确判定为自动成功或自动失败。

以下情况通常不要求检定：

- 角色拥有充足时间、工具和能力完成普通工作；
- 行动没有压力且失败不会产生任何后果；
- 玩家只是观察公开可见的信息；
- 同一事实已经通过可靠方式确认；
- Agent 只是希望延长场景；
- 玩家正在执行角色必然能够完成的日常动作。

## DC 设置

Agent 必须在掷骰前确定 DC，不得看到结果后临时提高或降低。

常用难度参考：

| 难度 | DC |
|---|---:|
| 非常容易 | 5 |
| 容易 | 10 |
| 中等 | 15 |
| 困难 | 20 |
| 非常困难 | 25 |
| 近乎不可能 | 30 |

玩家采用优秀方法、合适工具或获得有效帮助时，可以：

- 降低 DC；
- 获得优势；
- 自动获得部分信息；
- 避免某些失败后果。

玩家采用明显不利的方法时，可以：

- 提高 DC；
- 获得劣势；
- 增加耗时或资源消耗。

不得同时为了同一个理由既提高 DC 又给予劣势。

## 优势与劣势

- 优势和劣势采用 D&D 2024 默认规则。
- 多个优势来源不会叠加为更多骰子。
- 多个劣势来源不会叠加为更多骰子。
- 同时存在优势和劣势时相互抵消。
- Agent 必须说明产生优势或劣势的主要原因。

## 自然 20 与自然 1

- 攻击检定按 D&D 2024 规则处理自然 20 和自然 1。
- 死亡豁免按 D&D 2024 规则处理自然 20 和自然 1。
- 普通属性检定和技能检定不会仅因为自然 20 自动完成不可能的事情。
- 普通属性检定和技能检定不会仅因为自然 1 自动导致荒谬灾难。
- 极高或极低的原始骰可以影响叙事表现，但不能覆盖规则结果。

## 隐藏检定

适合隐藏处理的情况包括：

- 判断角色是否注意到未主动寻找的危险；
- NPC 是否察觉角色的谎言；
- 敌人是否发现角色踪迹；
- 玩家不知道某件事是否存在时的秘密搜索结果；
- 世界后台事件和敌对势力行动。

隐藏检定不得：

- 反复否定角色已经确认的证据；
- 用于强迫预设剧情发生；
- 在玩家提出合理行动后秘密取消其成果；
- 替代本应由玩家作出的决定。

隐藏检定必须记录：

- 骰点公式；
- DC 或对抗目标；
- 结果；
- 产生的实际影响；
- 对应事件 ID。

## 被动值

- 被动察觉用于角色没有主动搜索时，判断其是否注意到持续存在的细节或危险。
- 主动搜索时通常进行相应检定，而不是同时使用主动骰和被动值取更高。
- 其他被动技能只有在规则或场景确有需要时使用。
- Agent 不得使用隐藏检定反复绕过已经足够高的被动察觉。

## 失败推进

普通失败可以产生以下一种或多种后果：

- 花费更多时间；
- 消耗资源；
- 暴露位置；
- 引起 NPC 怀疑；
- 获得不完整信息；
- 成功但附带代价；
- 使敌对势力推进计划；
- 迫使玩家选择另一条路线。

失败不代表玩家角色突然表现得愚蠢。

核心任务不能因为一次普通失败永久无法继续。重要结论应存在其他线索、路线或代价更高的替代方案。

## 重复检定

只有满足以下条件之一时才允许对同一目标重新检定：

- 场景条件发生变化；
- 角色采用新的有效方法；
- 获得新工具或帮助；
- 花费显著额外时间；
- 承担新的风险或代价；
- 上一次检定只覆盖有限时间范围。

不得无代价连续掷骰直到成功。

## 关键骰点记录标准

出现以下任一情况时，写入当前会话的 `关键骰点（Key Rolls）`：

- 改变任务、关系或世界状态；
- 造成长期资源或生命后果；
- 获得或错失关键线索；
- 决定战斗、追逐或谈判的转折；
- 产生角色死亡或永久状态风险；
- 触发重要世界事件；
- 决定关键 NPC 的命运。

建议记录格式：

| 事件 ID | 行动者 | 检定 | 骰点公式 | DC/目标 | 结果 | 影响 |
|---|---|---|---|---|---|---|
| evt-0001 | pc-alen | 察觉 | 1d20+3=17 | DC 15 | 成功 | 发现道路边的鳞片 |
